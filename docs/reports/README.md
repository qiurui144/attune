# attune — one-shot reports / audits / baselines 归档

> 这里存放一次性产物:版本审计、GA verification、性能 baseline、E2E 报告。
> 长期 reference docs 在 `docs/` 顶层(per global CLAUDE.md §1.1.2 白名单);
> release notes SSOT 在 `RELEASE.md`(根);设计 spec 在 `docs/superpowers/specs/`。

## 文件命名

- `vX.Y.Z-<topic>-<purpose>.md` — 与具体 release 绑定的产物
- `<date>-<topic>.md` — 与日期绑定但跨 release 的产物

不再追加新条目时,**结论应入 RELEASE.md 对应版本节,文件可删**。

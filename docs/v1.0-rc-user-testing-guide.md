## 2. 安装步骤

### 2.1 Windows — NSIS exe（推荐）

1. 双击 `Attune_1.0.0_x64-setup.exe`
2. 若弹出 UAC 提示点击 **是**
3. 跟随 NSIS 向导，保持默认安装路径 `C:\Program Files\Attune`
4. 安装完成后勾选"启动 Attune"，点击 **完成**

`<screenshot needed>`

> 安装脚本自动检测 Ollama；若未安装会在 Wizard Step 3 提示。

### 2.2 Windows — MSI

适用于企业 Group Policy 批量部署：

```powershell
msiexec /i Attune_1.0.0_x64_en-US.msi /quiet
```

或直接双击 `.msi` 文件跟随向导。

### 2.3 Linux — DEB（Ubuntu / Debian）

```bash
# 安装
sudo dpkg -i Attune_1.0.0_amd64.deb
sudo apt-get install -f          # 解决依赖（如有缺失）

# 启动
attune                            # CLI 或从应用菜单搜索 "Attune"
```

`postinst` 脚本会自动安装 Ollama（如未安装）、拉取默认 embedding 模型。

### 2.4 Linux — AppImage（任意发行版）

```bash
chmod +x Attune_1.0.0_amd64.AppImage
./Attune_1.0.0_amd64.AppImage
```

AppImage 不含安装钩子，首次启动 Wizard Step 3 会引导安装 Ollama。

### 2.5 Linux — RPM（Fedora / RHEL）

```bash
sudo dnf install ./Attune-1.0.0-1.x86_64.rpm
```

行为与 .deb 完全一致（共用 4 个 hook 脚本）。

## 3. 第一次启动 — Wizard 5 步

首次启动自动进入 5 步 Wizard。跳过将进入"演示模式"（Chat 禁用，索引功能可用）。

`<screenshot needed>` — Wizard Step 1 欢迎页

### 3.1 Step 1 — 欢迎

点击**开始设置**进入 Step 2。

### 3.2 Step 2 — 主密码

| 字段 | 要求 |
|------|------|
| 主密码 | ≥12 字符，包含字母 + 数字 |
| 再次输入 | 必须一致 |

`<screenshot needed>` — Step 2 密码设置

**重要**：设置完成后系统会自动下载恢复密钥文件 `attune-recovery-key.txt`，请**立即保存**到安全位置。忘记主密码唯一恢复途径是该文件。

### 3.3 Step 3 — AI 大脑（LLM）

推荐选项（按优先级）：

| 选项 | 适合 |
|------|------|
| **★ Attune Pro Gateway**（推荐） | 注册账号即用，自动管理 token 配额 |
| **BYOK — OpenAI / Anthropic / Gemini** | 已有付费 API key 的用户 |
| **DeepSeek / Qwen / 自定义 OpenAI 兼容** | 其他 API |
| **本地 Ollama** | K3 一体机 / 高级用户 |
| **暂不配置（演示模式）** | 先体验界面，Chat 暂不可用 |

填写后点击**测试连接**，等待"连接成功"提示。

`<screenshot needed>` — Step 3 LLM 配置 + 测试结果

### 3.4 Step 4 — 硬件检测

自动检测 RAM / GPU / NPU 并推荐 embedding 模型：

| RAM | 推荐模型 |
|-----|---------|
| ≥16 GB + 独显/NPU | `bge-m3`（Ollama） |
| 16–32 GB + 核显 | `bge-base`（ORT） |
| 8–16 GB | `bge-small`（ORT） |
| <8 GB | `bge-small`（ORT，CPU 模式） |

检查推荐是否与你的硬件匹配，可手动调整，点击**确认**。

`<screenshot needed>` — Step 4 硬件检测结果

### 3.5 Step 5 — 数据源

三选一（也可跳过，之后在设置中添加）：

- **绑定文件夹**：选择本地目录自动监听
- **导入 .vault-profile**：从旧设备备份恢复
- **跳过，先看看**

点击**完成 · 进入 Attune**。

## 8. 故障排查

| 症状 | 排查步骤 |
|------|---------|
| Chat 不响应 / "AI 未配置" | Settings → AI 大脑 → 重新测试连接；检查 API key 是否有效 |
| Vault 锁住无法解锁 | 确认使用设置时的主密码（区分大小写）；如忘记密码，用 `attune-recovery-key.txt` 重置 |
| OCR 返回为空 / 错误 | 检查图片清晰度（≥300 dpi 建议）；确认 scene 选择正确 |
| OCR / ASR 极慢 | Settings → 硬件 → 查看 RAM 和 embedding 模型分配；8 GB RAM 下 OCR 较慢属正常 |
| Sources 上传后状态卡在"处理中" | 检查磁盘剩余空间（至少 2 GB）；重启应用后重试 |
| Linux deb 安装报依赖错误 | `sudo apt-get install -f` 自动补依赖 |
| Linux RPM GPG 验证失败 | `sudo rpm --import https://qiurui144.github.io/attune/attune-archive-keyring.gpg` |
| Ollama 未找到但已安装 | 确认 Ollama 服务在运行：`ollama list`；如未运行：`ollama serve &` |
| Windows 安装后无法启动（缺 DLL） | 安装 [Visual C++ Redistributable 2019+](https://aka.ms/vs/17/release/vc_redist.x64.exe) |
| Windows 安装报"Windows 保护了你的电脑" | 点击"更多信息" → "仍然运行" — rc.2 代码签名正在办理，GA 前修复 |

## 11. 卸载

### Windows

**开始菜单** → 搜索 "Attune" → 右键"卸载"；  
或**控制面板** → 程序 → 卸载 Attune。

> 卸载不删除用户数据（`%APPDATA%\attune`）和 Ollama；用户独立决定是否清除。

手动清数据：

```powershell
Remove-Item -Recurse -Force "$env:APPDATA\attune"
Remove-Item -Recurse -Force "$env:APPDATA\npu-vault"
```

### Linux — deb

```bash
sudo apt remove attune        # 保留用户数据
sudo apt purge attune         # 同上（数据仍保留，用户独立决定）

# 完全清数据
rm -rf ~/.local/share/attune ~/.config/npu-vault
```

### Linux — rpm

```bash
sudo dnf remove attune
rm -rf ~/.local/share/attune ~/.config/npu-vault
```

### Linux — AppImage

```bash
rm Attune_1.0.0_amd64.AppImage   # 删文件即卸载
rm -rf ~/.local/share/attune ~/.config/npu-vault
```

### 同时卸载 Ollama（可选，独立决定）

```bash
sudo systemctl disable --now ollama
sudo rm /usr/local/bin/ollama
rm -rf ~/.ollama
```

## 12. 5/25 GA 升级路径

**rc.2 → v1.0.0 GA（5/25 发布）**

| 升级方式 | 条件 | 步骤 |
|---------|------|------|
| **应用内自动升级**（推荐） | Tauri auto-updater 已配置（`#80`） | 应用顶栏出现"有新版本"提示 → 点击升级 → 重启 |
| **手动覆盖安装** | 自动升级未出现时 | 从 [Releases](https://github.com/qiurui144/attune/releases) 下载 GA 安装包，覆盖安装（数据不丢失） |
| **包管理器升级** | 已用 APT / DNF / WinGet | `sudo apt upgrade attune` / `sudo dnf upgrade attune` / `winget upgrade qiurui144.Attune` |

**升级前不需要手动备份**：rc.2 数据格式与 GA 完全兼容，vault schema 幂等迁移。

> 如果 auto-updater（`#80`）在 GA 前未合入，升级只需手动下载新安装包覆盖安装，用户数据 `~/.local/share/attune` 自动保留。

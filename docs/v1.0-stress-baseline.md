# v1.0 Stress Test Baseline

## 测试环境

- 平台: Linux x86_64
- 模式: `cargo test --release --test <name> -- --nocapture --ignored`
- Embedding: 64-dim deterministic mock vector（不调 Ollama，seed 确定）
- 日期: 2026-05-23

## Crash Recovery（`crash_recovery_test`）

| 测试 | 结果 | 说明 |
|------|------|------|
| WAL 模式 drop 后重开 | PASS | 5 items 全部保留 |
| 多次 open/close 幂等 | PASS | 3 次 open/close，5 items 完整 |
| reindex drop 后重开 | PASS | 10 embed 任务跨 drop 持久化 |
| embed queue drop 后重开 | PASS | 8 任务 dequeue 并 mark done |

## 并发 Stress（`concurrent_stress_test`）

| 测试 | 结果 | 数据点 |
|------|------|--------|
| 10 线程读写无死锁 | PASS | 5 writer × 20 items = 100 items，无死锁 |
| 5 embed + 5 search 并发无 race | PASS | 1000 向量写入，0 错误 |
| 4 consumer exactly-once | PASS | 42 任务，无重复 dequeue |

## OOM 行为（`oom_behavior_test`）

| 测试 | 结果 | 数据点 |
|------|------|--------|
| 100 MB 大文件 ingest | PASS | ~100 GB → 261886 chunks，不 panic |
| vector index 满载搜索（9999 向量） | PASS | top-10 结果稳定，不 corrupt |
| 200 KB 超限内容 chunker | PASS | → 568 chunks，不 panic |
| 空/极短内容 edge case | PASS | 6 种边界情况，无 panic |

## VectorIndex capacity 注意事项

`VectorIndex::new(dims)` 内部调用 `usearch::Index::reserve(10_000)`。
写入超过 10000 个向量需要额外 reserve（当前无公开 API）。
大规模 stress test 的向量写入上限为 9999，超出需在 `vectors.rs` 暴露 `pub fn reserve(n)`。

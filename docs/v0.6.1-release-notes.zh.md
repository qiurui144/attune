# Attune v0.6.1 — 发布说明

_发布日期：2026-04-30_

## 一句话

Attune v0.6.1 是**边界收敛 + 形态分裂 patch**。统一 OSS × Pro 在所有 README/spec 文档的叙事，引入 `FormFactor` 让 K3 一体机镜像默认走本地 Ollama 而笔电保持 v0.6.0 GA 的 "远端 token 默认" 行为，并修了 `rustls-webpki` 3 个 RUSTSEC 漏洞。**0 breaking change** — v0.6.0 笔电用户体验完全不变。

## 新增

- **`FormFactor` 形态轴**（`platform/mod.rs`）：新增 `Laptop` / `K3Appliance` / `Server` / `Unknown` enum 到 `HardwareProfile`。检测顺序：`ATTUNE_FORM_FACTOR` env var override > Linux DMI 关键字（`K3` / `Jetson`）> 默认 `Laptop`。
- **`default_settings` 形态分裂**（`routes/settings.rs`）：`Laptop` / `Server` / `Unknown` 保持 v0.6.0 默认 `llm.provider = "openai_compat"`（远端 token，wizard 引导填 API key）；`K3Appliance` 默认 `provider: "ollama"` + `endpoint: "http://localhost:11434/v1"` + `model: "qwen2.5:3b"`，配合 K3 镜像预装本地 LLM。
- **`/status/diagnostics` 暴露 `form_factor` + `prefers_local_llm`** 字段，让桌面 wizard 显示对应默认。
- **Wizard Step 3（选 AI 大脑）形态感知**（`Step3LLM.tsx`）：K3 主机看到 Ollama 卡片为 ★ 推荐；笔电主机看到 Cloud 卡片为 ★ 推荐。非推荐卡片仍可点选（dashed border 标记）。
- **`PREFERRED_MODELS` doc-comment 精确化**（`llm.rs`）：明确说明这是 Ollama auto-detect 的 fallback 列表，**不是默认行为**。仅在用户在 wizard 主动选 Ollama 模式或 `settings.llm.provider = "ollama"` 时使用。

## 文档收敛（OSS × Pro 边界）

修复 6 处文档与代码漂移，让代码、CLAUDE.md、对外 README、4-17 定位 spec 收敛到同一真相：

- README（`/README.md` + `.zh.md` + `rust/README.md` + `rust/README.zh.md`）：顶部 tagline + "Who it's for" 表格改为"任何领域个人知识工作者"，并新增 **Industry users** 段落指向 `attune-pro/<vertical>-pro` 插件包。
- `docs/oss-pro-strategy.md`（+ `.zh.md`）：§3 Decision 3 Lite 行不再声明 "all 4 builtin basic plugins"；§4.1 OSS 范围加 Persona system 行（4 通用角色，🟡 v0.7+）；§4.2 Pro 范围加 Industry personas 行（Lawyer / Doctor / PatentAgent 等）。
- `docs/superpowers/specs/2026-04-17-product-positioning-design.md`：顶部 banner 说明 §5 用户/场景已按 oss-pro-strategy v2 迁移到 `attune-pro/docs/specs/<vertical>-pro-vertical-workflow-design.md`；§3 LLM 段表述按形态分裂修订。
- `CLAUDE.md`：新增「Git 分支管理标准（GitFlow Lite）」段，覆盖两条长期分支铁律 / develop→main 两类时机 / Tag 双轨制 / `--no-ff` 强制 / `--first-parent` 视角的 "main 无非 merge commit" 不变量。
- `attune-pro/docs/specs/`：新增 `law-pro` / `patent-pro` / `academic-pro` 三份 vertical workflow design spec（共 508 行），每份按 OSS-Pro 5 层价值矩阵（L1-L5）映射。

## 安全

- `rustls-webpki` 0.103.10 → 0.103.13 — 修了 3 个 CVE：
  - RUSTSEC-2026-0099（name constraints 错误接受 wildcard names）
  - RUSTSEC-2026-0098（URI names 的 name constraints 错误接受）
  - RUSTSEC-2026-0104（CRL 解析可达 panic）
- 0 个剩余 vuln；8 个 unmaintained 依赖 warning（core2 / fxhash / instant / paste / rustls-pemfile / lru — 非安全漏洞）。

## 测试

- 622 个测试通过（lib + integration + binary），0 failed，3 ignored。从 v0.6.0 GA 报告的 "237+" 上调 — 这个数字包括 `attune-core` lib（535）+ `attune-server` lib（5）+ `attune-server-headless`（3）+ 16 个 integration 套件。
- 4 个新 `FormFactor` 单元测试在 `platform/mod.rs::tests`，覆盖 env-var override / default / `prefers_local_llm` / `HardwareProfile::detect` 集成。
- 4 个新 `default_settings` 单元测试在 `routes/settings.rs::tests`，验证 Laptop → 远端 token、K3 → 本地 Ollama、Server/Unknown → fallback 远端、非 LLM settings 在所有形态下保持不变。

## 兼容性

- v0.6.0 笔电用户：**0 行为变化**。Laptop/Server/Unknown 形态下 `default_settings` 输出与 v0.6.0 完全相同（远端 token 默认）。
- K3 一体机镜像构建者：在 systemd service unit 中设 `ATTUNE_FORM_FACTOR=k3`（或依赖 DMI 关键字检测），wizard 会默认走本地 Ollama + 预装 `qwen2.5:3b`。
- 0 数据库 schema 变更。0 vault 迁移需求。

### Tab 1 — `attune` (free OSS,永久免费)

**给谁**:

- 学生 / 独立开发者 / 研究者 / 作者 / AI 重度用户。
- 任何愿意"把硬盘里的知识接到 LLM"且**重视隐私**的个人知识工作者。
- 律师 / 患者 / 医生 / 学者也能用 — 但行业能力需另装 attune-pro。

**做什么**:

| 能力 | 详细 |
|------|------|
| 🔒 加密 vault | DEK + AES-256-GCM + Argon2id KDF + Sidecar 表模式,密码错误锁仓即数据无法读取 |
| 🔍 RAG 混合检索 | BM25 (tantivy + 中文 jieba) + 向量 (usearch HNSW f16) + RRF 融合 + J3 显式 min_score |
| 💬 内置 Chat | RAG 强约束 prompt + 二次检索 + 引用 deep-link (breadcrumb + chunk offset + confidence 1-5) |
| 📥 5 内置 ingest connector | Local 文件夹监听 / Email IMAP / WebDAV / RSS / Telegram (scaffold) |
| 🌐 浏览捕获 | Chrome 扩展 V3 默认 opt-out + HARD_BLACKLIST + G5 隐私面板 + 自动 bookmark staging |
| 🧠 通用 agent | Memory consolidation L2→L3 / Chat Reliability post-hoc / Internal Linker(跨 item 链接) |
| 🔁 主动进化 skill | SkillClaw 风格:失败信号 → LLM 扩展词 → 静默生效 |
| 🛡️ 隐私三层 | L0 per-file 永不离机 / L1 12 类 PII 反向占位符 + outbound 审计 / L3 LLM-based 语义脱敏(K3 tier) |
| 🌍 多语言 | 中英文 i18n 双 locale 一致,key 集合等价(per CLAUDE.md i18n 规范) |
| 📦 跨平台分发 | Win NSIS / Win MSI / Linux deb / RPM / AppImage(v0.7.0 实测) |

**不做什么**(明确边界):

- ❌ 不内置任何行业插件(律师 / 患者 / 医生 / 患者 / 学者 / 售前 / 工程师 / 专利代理) — 全部在 attune-pro per v0.6.0-rc.2 边界瘦身
- ❌ 不在浏览器扩展里给 web AI 注入 prompt 前缀(产品 2026-04-12 转向**内置 Chat + RAG**,cleanup-r15 删除注入路径)
- ❌ 不提供云存储(数据 100% 本地 + 用户自己的 NAS via WebDAV)
- ❌ 不提供 MCP backbone 给第三方 AI(至少 v0.7 不做)
- ❌ 不本地预装 LLM 模型(LLM 走远端 token,本地仅捆绑必要底座:embedding / rerank / ASR / OCR + Ollama runtime;K3 一体机镜像例外)

**价格**:¥0 永久免费。Apache-2.0。

### 3.6 Office Helper — OCR / ASR 实际能力

来源:`rust/crates/attune-server/src/routes/office.rs` + `bin/headless.rs` + `state.rs`。

**OCR**:

- 引擎:**PP-OCRv5 mobile (ONNX)**,~16 MB 模型
- 一键引导:`attune deploy` 启动时自动下载到 `~/.local/share/attune/models/ocr/`
- 输入:PDF (via `pdftoppm`) / 图片(JPG/PNG/HEIC/WEBP)
- **5 个场景**(v1.0 GA 覆盖):
  1. 票据 / 发票
  2. 名片
  3. 表格(身份证 / 文档表)
  4. 合同 / 文档全文 OCR(配合 chunker)
  5. 截图(微信 / 网页截图 → 文本提取)

**ASR**:

- 引擎:**whisper.cpp** binary + Rust subprocess
- 默认模型:`whisper-small-q8`(480 MB,实测中文 WER < 20% 满足)
- 高 tier 推荐:`whisper-large-v3-turbo Q5`(中文 WER 5-7%,2024-10 sota)

## §5 重点应用场景 5 个

> 每个场景写成"问题 → attune 怎么做 → 用户得到什么",可直接用作 blog / landing solutions page 内容。

### 场景 1 — 律师收件(law-pro 旗舰场景)

**问题**:律师每天收到当事人发来的微信 / 邮件 / 借条照片 / 银行流水 PDF,几十个分散在云盘和聊天软件里。每个新案件要花 1-2 小时人工整理证据 + 算利息 + 查时效 + 写陈述。

**attune 怎么做**(实际可执行链):

1. **Email connector** 自动拉取 `客户1@email.com` 邮箱中带"借条/流水/合同"附件的邮件,落入 Project `劳动争议-张三案`
2. **PP-OCRv5** 把借条照片 + 流水 PDF OCR 成文本,自动识别为"借条"/"银行流水"证据类型
3. **fact_extractor**(LLM,~2000 token)抽取本金 / 利率 / 起始日 / 还款记录 — **每个字段带原文依据**(grounded),ungrounded 字段标记 `need_lawyer_confirm`
4. **civil_loan_agent**(零 LLM,纯公式)按《民间借贷规定》§680 + LPR×4 上限算应付利息,**audit trail 完整**(律师能复现每步算术)
5. **limitation_agent** 按《民法典》§188 检查诉讼时效是否届满(3 年普通 / 20 年最长),中断事由路由给律师人工确认
6. **bank_aggregator_agent** 把 PDF 流水按收/付/日期聚合,与借条本金做交叉验证(hallucination guard)
7. **evidence_chain_agent** 描述证据间印证 / 矛盾 / 重叠 + 标记缺口(如"缺收据印证 2023-05-10 转账")
8. **Chat** 内提问"这个案件可以起诉吗?预计能追回多少?"— RAG 检索 agent 全部 audit trail + LLM 综述,**回答带 confidence 1-5 + 每个引用点回原文 chunk**

**用户得到什么**:

- 30 分钟从邮件入库到出 demand letter 草稿(节省 60-90 分钟)
- 算术 100% 可验证(deterministic gate)
- 法律判断显式路由给律师(framework 不假装能判)
- 数据 100% 在律师自己电脑/NAS,**当事人隐私零云端**(微信流水从来不出本机)

**对应 agent**:civil_loan / bank_aggregator / fact_extractor / limitation / evidence_chain(5 个全部命中)。

### 场景 2 — 知识工作者(本地 RAG + 跨 item linking + 主动进化记忆)

**问题**:研究者 / 高级开发者 / 作者 / AI 重度用户硬盘上有上千个 markdown / PDF / 论文 / 对话存档,需要时翻不到。用 Notion AI / ChatGPT Memory 但担心隐私 + 数据离开自己设备。

**attune 怎么做**:

1. **Local 文件夹监听**:把 `~/Documents/notes` + `~/Research/papers` 加入 vault,自动 OCR + chunk + 双索引(BM25 + 向量)
2. **加密 vault**:Argon2id + AES-256-GCM,密码错误锁仓即数据无法读取
3. **混合检索**:同时跑 tantivy(中文 jieba 分词)+ usearch HNSW,RRF 融合
4. **内置 Chat**:RAG + 强约束 prompt + 二次检索 + 引用 deep-link(breadcrumb + chunk offset + confidence 1-5),点回原文 reader modal
5. **Internal Linker agent**:跨 item 找"这篇论文引用过那本书的哪个章节",自动建立内部链接
6. **Memory Consolidation agent**:L2 短期记忆 → L3 长期记忆固化,SkillClaw 风格静默自学(失败信号 → LLM 扩展词 → 静默生效)
7. **Chrome 扩展**:浏览捕获默认 opt-out,只在用户主动 bookmark 时自动 staging
8. **跨设备**:Vault 文件夹放在 WebDAV(自己 NAS / Synology / Nextcloud),笔电 + 台式机同步;DEK 永不离机

**用户得到什么**:

- **0 知识泄漏到云端**(LLM 推理走 BYOK 或本地 Ollama,但 RAG 召回都在本机做完)
- 越用越懂自己研究方向(Memory + Linker 自学)
- 跨设备无缝(WebDAV 加密 blob)
- 切换雇主 / 项目时直接迁移 vault 文件夹

**对应 agent**:Memory consolidation / Chat Reliability / Linker(3 个 OSS 域 agent 全部命中)。

### 场景 3 — 研究者(Email + RSS 订阅 → 自动索引 + 链接)

**问题**:研究者订阅 arXiv RSS + Nature / Science / IEEE 邮件推送,每天 50-200 条更新,人工读完 + 归档 + 找跨 paper 关联根本不可能。

**attune 怎么做**:

1. **RSS connector**:订阅 arXiv 子分类 RSS,每 N 小时拉一次,新条目自动入 vault(标 `corpus_domain = research`)
2. **Email connector**:Nature 推送邮件 IMAP 拉取,正文 + 附件 PDF 全量 OCR 入库
3. **F-Pro Cross-domain pollution defense**:`[领域: research]` chunk prefix + cross-domain penalty(0.4),避免"transformer" 召回 Java algorithm 文档
4. **Linker agent**:自动找新 paper 与既有 vault 中的 paper 的概念链接(citation 网络 + topic 相似度)
5. **Skill evolution**:用户搜过 "diffusion attention head 可解释性" 失败 → 后台 LLM 扩展词为 "interpretability of attention in diffusion models" → 下次该 query 命中
6. **Chat**:"上周 arXiv 出的 diffusion 论文里哪些用了 RAG 增强?帮我总结" — 引用每个 paper 的 abstract chunk

**用户得到什么**:

- 50-200 条 / 天的 firehose 沉淀成可搜可问的知识库
- 跨 paper 关联自动建立(不靠人工 tag)
- 用户搜索失败信号自动反推 skill 进化
- 数据本地(arXiv 是开源 OK,但用户的注解 / 笔记 / private comments 一定本地)

**对应 connector**:RSS + Email(2 个);agent:Linker + skill_evolution。

### 场景 4 — 个人秘书(Office helper 票据 / 名片 / 表格 OCR)

**问题**:小公司主 / 自由职业者 / 个人执业者每月报销 / 客户名片 / 收据要手工录入财务系统,耗时且易错。

**attune 怎么做**:

1. **iOS Shortcut / 截图扫描**:把票据照片拖入 attune
2. **PP-OCRv5 mobile**:5 场景 OCR(票据 / 名片 / 表格 / 合同 / 截图)— ~16 MB 模型,本机推理
3. **结构化抽取**:模板化字段(发票号 / 金额 / 日期 / 商家),OCR 文本 + LLM 抽取结合
4. **Vault 加密落盘**:票据 metadata 加密,原图也加密(防云盘泄漏)
5. **导出**:每月一次性 export CSV 给会计 — DEK 不出机,只 export 选定数据

**用户得到什么**:

- 月底 30 分钟搞定 50 张票据录入(原本 3-4 小时)
- 名片 OCR 自动入"联系人"vault
- 客户信息隐私(老板 / 客户名片不进任何 cloud OCR API)

**对应能力**:PP-OCRv5(5 场景)+ 加密 vault + 导出工具链。

### 场景 5 — 私域知识库(Vault 加密 / 离线 / 跨设备)

**问题**:咨询师 / 律师 / 医生 / 心理咨询师等执业用户,客户档案高度敏感(医疗记录 / 律所卷宗 / 心理咨询记录)。怕用 Notion / Google Drive — 数据在别人服务器。怕用本地 Markdown — 没有索引 / 没有 Chat / 没法跨设备。

**attune 怎么做**:

1. **加密 vault**:Argon2id + AES-256-GCM + Sidecar 表模式 — 即使有人物理拿走你的硬盘,没密码也读不出
2. **L0 隐私级别**:per-file 标记"永不离机",该文件 chunk **强制走本地 LLM**(Ollama),不发任何云
3. **L1 默认**:12 类 PII(身份证 ISO 7064 校验 / 手机号 / 邮箱 / 8 vendor API key)用 `[KIND_N]` 反向占位符脱敏后才能进 LLM prompt;outbound 全量 CSV 审计可导出合规
4. **L3(K3 tier)**:LLM-based 语义脱敏 — "客户提到他在某药企任 VP" 自动改写成"客户提到他在某行业任高管"
5. **WebDAV 同步**:存自己 NAS,只同步加密 blob,DEK 永不离机;笔电 + 出差备机 + 办公室台式机三设备同步
6. **离线 mode**:全程不联网也能用 — 本地 Ollama + 本地 OCR / ASR,只 RAG + Chat,完整工作流不依赖云

**用户得到什么**:

- 物理隔离级隐私(密码 + 设备双因素)
- 跨设备无缝(WebDAV 加密 blob)
- 合规审计就绪(outbound CSV)
- 离线可用(出差 / 国际航班 / 安全屋场景)

**对应能力**:Vault 三层加密 + L0/L1/L3 隐私模型 + WebDAV + Ollama 本地 LLM。

## §7 关键 differentiation 句(2-3 句)

> 用于:README hero / landing hero / 名片背面 / 推特置顶 / PR 文案。中英双语。每句独立可用。

### 句 1 — 隐私 / 本地优先(对照 ChatGPT / Notion AI)

**中**:不像 ChatGPT 把对话存在 OpenAI 服务器,也不像 Notion AI 把文档丢到 SaaS 数据库 — Attune 把所有数据用 Argon2id + AES-256-GCM 加密在你自己设备上,LLM 推理默认本地,云端按需,**密码错误锁仓即数据无法读取**。

**EN**: Unlike ChatGPT which stores conversations on OpenAI's servers, or Notion AI which lives in a SaaS database — Attune encrypts every byte with Argon2id + AES-256-GCM on **your** device. Local-first inference, cloud-on-demand. Wrong password = sealed vault, data unreadable.

### 句 2 — 律师级 Reliability framework(对照普通 AI 助手)

**中**:别的 AI "帮你算利息" 没人能验证它有没有算错;Attune 的 9 个律师 deterministic agent 用 **1.00 硬门 CI-blocking** 验证算术,1 个 LLM 抽取 agent 用 micro_F1 = 0.9828 + ratchet-upward floor 验证抽取,**框架显式声明"不证明法律正确性,只证明计算正确性,法律判断显式路由给律师"**。

**EN**: Other AIs "calculate your interest" — but you can't verify they didn't hallucinate. Attune's 9 deterministic legal agents pass a **1.00-exact CI-blocking gate**; the LLM extractor enforces a `micro_F1 ≥ 0.85` ratchet-upward floor (currently **0.9828**). The framework explicitly states: *we certify computational correctness, route legal judgement to the lawyer.*

### 句 3 — 混合智能 / 分层成本透明

**中**:Attune 把每次计算的钱写在 UI 上 — 🆓 BM25 检索零成本秒级 / ⚡ embedding / OCR / ASR 本地算力秒级 / 💰 LLM 调用时间 + token 显式预估,**永不后台偷跑 LLM**。建库阶段只跑前两档,深度分析等用户敲回车。

**EN**: Attune writes the cost of every computation directly into the UI — 🆓 BM25 retrieval (free, ms) / ⚡ local embedding/OCR/ASR (your hardware, sec) / 💰 LLM calls (time + tokens estimated upfront). **The LLM never runs in the background.** Indexing stops at "searchable + 150-char summary"; deep analysis waits for the user to press Enter.

### 候选(备用 — 短句版,可塞 hero 一行)

- **中**:"私密、本地优先、越用越懂你的专业。"
- **EN**: "Private. Local-first. Increasingly attuned to your expertise."
- **中**:"Your knowledge stays. The LLM helps when you ask."
- **EN**: "Your data stays. The AI assists on request."
